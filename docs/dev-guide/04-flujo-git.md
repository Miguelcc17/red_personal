# 04 - Flujo de Git

## Ramas
- `main`: Código estable y listo para producción.
- `feature/*`: Desarrollo de nuevas funcionalidades.
- `fix/*`: Corrección de errores.

## Commits
- Usar mensajes descriptivos en presente imperativo (ej. "Add person creation flow").
- Referenciar tickets o issues si existen.

## Pull Requests
- Deben incluir una descripción de los cambios.
- Pasar pruebas básicas antes de solicitar revisión.
